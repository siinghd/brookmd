# flux-md Kotlin bindings (EXPERIMENTAL)

First-party Kotlin/Android bindings for the flux-md streaming Markdown parser,
generated by [uniffi](https://mozilla.github.io/uniffi-rs/) from the
`crates/flux-md-ffi` crate. The bindings expose a `FluxSession` that streams
Markdown and returns the JSON **wire strings** defined in
`crates/flux-md-core/WIRE.md` (contract v1.0.0) — byte-identical to the
WASM/JS boundary.

> **Status: EXPERIMENTAL.** The generated bindings compile and pass the
> wire-golden smoke tests on the **JVM host** in CI (`bindings-build.yml`). They
> are **not yet device-validated** on Android and **not published** to Maven.

## What's here

| Path | What it is |
| --- | --- |
| `fluxmd/` | `com.android.library` module (`io.github.siinghd.fluxmd`, `minSdk 24`). Packages the generated `.kt` plus per-ABI `jniLibs/<abi>/libflux_md_ffi.so`. JNA `@aar` runtime dep. |
| `fluxmd/src/main/kotlin/io/github/siinghd/fluxmd/flux_md_ffi.kt` | **Generated** uniffi bindings (do not hand-edit — regenerate). |
| `host-test/` | Standalone `kotlin("jvm")` build that compiles the *same* generated `.kt` (via a `srcDir` reference) and runs the wire goldens against the host `.so` using the plain JNA jar. |
| `scripts/generate.sh` | Regenerates the bindings from the FFI crate (idempotent). |

The two Gradle builds are **decoupled on purpose**: the JVM host test needs
neither the Android Gradle Plugin nor an Android SDK, so a broken/absent Android
toolchain can never mask a wire regression.

## API

```kotlin
import io.github.siinghd.fluxmd.FluxSession
import io.github.siinghd.fluxmd.FluxConfig

val session = FluxSession()                    // library defaults
// or: FluxSession.newWithConfig(FluxConfig(gfmAlerts = true, blockData = true))

val patch = session.append("# Hello\n\n")       // JSON Patch string (WIRE.md §2)
val tail  = session.finalize()                  // final JSON Patch
val doc   = session.allBlocks()                 // whole document as JSON Block[]
session.reset()                                 // reuse the session, ids restart at 0
```

`append` / `finalize` / `allBlocks` return the exact wire bytes; decode them with
any JSON parser. `FluxConfig` field defaults mirror the crate (`gfmAutolinks` and
`gfmAlerts` **on**, everything else off).

## Consuming

**As source (today):** depend on the `fluxmd` module in a composite/settings
`includeBuild`, and supply `libflux_md_ffi.so` per ABI under
`fluxmd/src/main/jniLibs/<abi>/` (build them with the RN package's
`scripts/build-android.sh`, which cross-compiles the same crate with the NDK).

**As an AAR:** run `gradle -p fluxmd assembleRelease` after staging the `.so`
files; the resulting AAR bundles the bindings, the JNA `@aar`, and the native
libraries.

**Not on Maven yet** — publishing is a follow-up once the bindings are
device-validated.

### JNA note

uniffi's generated Kotlin uses JNA (`Native.register(..., "flux_md_ffi")`).
On Android the **`net.java.dev.jna:jna:5.14.0@aar`** classifier is required (it
ships the JNI dispatch libraries). On the plain JVM host test use the **normal
jar** (`net.java.dev.jna:jna:5.14.0`) and point JNA at the library directory
with `-Djna.library.path=<dir containing libflux_md_ffi.so>`.

## Regenerating

```sh
bindings/kotlin/scripts/generate.sh
```

This builds the crate's release staticlib and runs the **in-crate**
`uniffi-bindgen` (pinned to `uniffi = "=0.31.0"` — an external bindgen of another
version produces incompatible bindings), writing
`fluxmd/src/main/kotlin/io/github/siinghd/fluxmd/flux_md_ffi.kt`. Commit the
diff; CI regenerates and `git diff --exit-code`s it as a drift check.

> Bindings are read from the release **staticlib** (`.a`), not the `.so`: the
> release `.so` is `strip = true`, which removes the uniffi metadata symbols the
> bindgen needs. The `.a` retains them.

## Running the host test locally

```sh
# 1. build the host native library
( cd crates/flux-md-ffi && cargo build --release )
# 2. run the JVM golden test against it (needs a JDK + Gradle)
gradle -p bindings/kotlin/host-test test \
  -Djna.library.path="$PWD/crates/flux-md-ffi/target/release"
```

> The pinned Gradle/AGP/Kotlin versions target a modern toolchain; if they clash
> with your Gradle, adjust them in the `build.gradle.kts` files.
